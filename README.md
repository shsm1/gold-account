# 积小金 - 积存金记账管理APP

一款轻量级移动端黄金投资记账应用，专为个人积存金投资者设计。支持实时金价追踪、批次化管理、独立盈亏计算，数据完全本地存储，保护隐私安全。

**适用场景**：个人进行积存金、如意金等黄金投资时，辅助记录每笔交易、追踪持仓成本、计算浮动收益和已实现盈亏。

##  核心功能

### 📊 持仓总览
- **实时持仓统计** - 显示当前持仓克数、平均成本、持仓总价
- **账户余额计算** - 基于总投入金额，自动计算理论账户余额
- **浮动盈亏分析** - 输入实时金价，即时计算浮动盈亏和总市值
- **已实现盈亏** - 基于批次汇总的精确盈亏计算

### 📝 交易管理
- **买入记录** - 支持总价输入模式，自动计算单价；或手动输入克数、单价
- **卖出记录** - 支持从指定批次部分卖出，精确追踪每笔交易来源
- **手续费自动计算** - 根据设置的费率自动计算手续费，支持手动调整
- **批次标签** - 每笔买入自动生成独立批次，便于后续追踪

### 📦 批次管理
- **独立批次追踪** - 每个买入批次独立管理，清晰展示买入时间、成本、剩余克数
- **批次盈亏计算** - 每批次独立计算已实现盈亏，精确到每克
- **批次筛选** - 支持按状态筛选（全部/持仓中/已清仓）
- **低于现价筛选** - 一键筛选成本低于实时金价的持仓批次，并按成本从高到低排序
- **批次状态栏** - 显示符合条件的持仓总量，方便决策

### 🏦 如意金管理
- **独立账户管理** - 工行如意积存金独立记账，与普通积存金分开管理
- **提取记录** - 支持记录实物提取操作，不影响盈亏计算
- **批次追踪** - 如意金同样支持批次化管理，支持售出和提取操作

### 💰 实时金价
- **API 自动获取** - 接入国际金价 API，自动转换为人民币/克
- **美元汇率同步** - 实时获取 USD/CNY 汇率，确保价格准确
- **手动输入** - 支持手动输入金价，适用于网络异常或自定义价格
- **自动刷新** - 每分钟自动更新金价，保持数据时效性
- **金价筛选** - 支持筛选成本低于现价的持仓批次，快速识别盈利机会

### ⚙️ 参数设置
- **手续费费率** - 灵活设置买入/卖出费率（百分比）
- **总投入金额** - 手动设置总投资金额，用于计算账户余额
- **数据导入/导出** - JSON 格式备份与恢复，支持跨设备迁移
- **数据清空** - 一键清空所有交易记录

## 📐 核心算法说明

### 账户余额计算
```
账户余额 = 总投入金额 - 积存金净投入 - 如意金净投入

其中：
积存金净投入 = 累计买入金额(含手续费) - 累计卖出金额(扣手续费)
如意金净投入 = 如意金累计买入金额 - 如意金累计卖出金额(扣手续费)
```

### 批次成本计算（摊薄成本法）
```
实际成本/g = (买入单价 × 买入克数 + 买入手续费) / 买入克数
```

### 批次盈亏计算
```
已实现盈亏 = (卖出单价 - 批次实际成本) × 卖出克数 - 卖出手续费

总已实现盈亏 = 所有批次已实现盈亏之和
```

### 浮动盈亏计算
```
单批次浮动盈亏 = (当前金价 - 批次买入单价) × 批次剩余克数
总浮动盈亏 = ∑ 所有持仓批次的浮动盈亏

总市值 = 当前金价 × 总持仓克数
```

### 如意金摊薄成本
```
采用摊薄成本法：
- 买入时：累加总成本和持仓量
- 卖出时：按比例减少总成本和持仓量，计算盈亏
- 提取时：仅减少持仓量和总成本，不计算盈亏

平均成本 = 总成本 / 总持仓量
```

## 💾 数据管理

### 数据存储
- 使用 **IndexedDB** 本地数据库，数据完全存储在设备本地
- 支持离线使用，无需网络连接（除实时金价功能外）
- 数据持久化，关闭应用后数据不丢失
- 两个独立数据表：`transactions`（积存金）和 `icbc-transactions`（如意金）

### 数据导出
- **导出格式**：JSON 文件
- **文件名**：`积存金记账_YYYY-MM-DD.json`
- **保存位置**：
  - **浏览器环境**: 浏览器默认下载目录
  - **Android App**: Documents 文档目录（可在文件管理器的"文档"文件夹中查找）
- **导出内容**：包含所有交易记录的完整数据

### 数据导入
- 支持从 JSON 文件恢复数据
- 导入时自动合并现有数据
- 可用于跨设备数据迁移或备份恢复
- 文件格式验证，确保数据安全

## 🔒 隐私与安全
